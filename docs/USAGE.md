# AI 图片管理系统 - 使用说明

## 一、环境准备

在开始使用 AI 图片管理系统之前，请确保你的开发环境已安装以下工具：

- **Node.js 18+** — 前端构建工具运行环境，前往 [https://nodejs.org](https://nodejs.org) 下载安装
- **Rust** — Tauri 后端编译所需，通过 [rustup](https://rustup.rs) 安装：访问 [https://rustup.rs](https://rustup.rs) 下载并运行安装脚本
- **MSVC Build Tools 2019+**（Windows）或 **Xcode Command Line Tools**（macOS）— C/C++ 编译工具链
  - Windows：安装 [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)，勾选"使用 C++ 的桌面开发"工作负载
  - macOS：运行 `xcode-select --install`
- **npm** 或 **pnpm** — 包管理器（Node.js 安装时自带 npm）

## 二、安装与启动

```bash
# 1. 进入项目目录
cd ai-image-manager

# 2. 安装依赖
npm install

# 3. 安装额外的 UI 组件依赖
npm install @radix-ui/react-switch @radix-ui/react-slider

# 4. 启动开发服务器
npm run tauri:dev
```

> **注意：** 首次构建时，Rust 需要下载并编译所有依赖，通常需要 3-5 分钟。后续启动会快很多，因为依赖已缓存。

## 三、功能概览

AI 图片管理系统提供以下核心功能：

| 功能 | 说明 |
|------|------|
| 图片导入 | 支持拖拽或点击选择文件导入 |
| 自动预分类 | 根据文件名自动识别图片来源（ChatGPT、Gemini、Midjourney 等） |
| 标签管理 | 为图片添加自定义标签，支持自动补全 |
| 厂商/模型管理 | 管理 AI 厂商及其模型信息，支持多模型标记 |
| 水印检测与去除 | 自动检测常见 AI 平台水印，并提供去除功能 |
| 文件扫描 | 扫描指定文件夹，新文件手动导入或清理 |
| 归档管理 | 将图片按命名模板重命名并归档到指定目录 |
| 暗色模式 | 支持浅色、深色、跟随系统三种主题模式 |
| 键盘快捷键 | 提供常用操作的快捷键，提升操作效率 |
| 设置页面 | 集中管理应用配置，包括命名模板、水印设置、监控配置等 |

## 四、使用指南

### 4.1 导入图片

系统提供两种图片导入方式：

- **拖拽导入：** 将图片文件直接拖拽到待整理区域即可完成导入
- **点击上传：** 点击上传区域，在文件选择对话框中选择一个或多个图片文件

**支持的图片格式：** PNG、JPG、JPEG、WebP、GIF、BMP

导入后，系统会自动根据文件名分析图片来源。当识别置信度大于 0.5 时，会自动应用预分类结果，包括厂商、模型等信息。

### 4.2 编辑图片信息

点击任意图片卡片即可打开快速编辑弹窗，可编辑以下信息：

- **厂商** — 选择图片对应的 AI 厂商（如 OpenAI、Google、Midjourney 等）
- **模型** — 选择使用的 AI 模型，支持多选并可指定主模型
- **Prompt** — 输入生成该图片时使用的提示词，支持从剪贴板直接粘贴（Ctrl+V）
- **标签** — 为图片添加分类标签（详见 4.3 节）

### 4.3 标签管理

标签系统帮助你对图片进行分类和组织：

- 在编辑弹窗的标签区域输入文字，按回车添加新标签
- 点击标签上的关闭按钮可删除标签
- 输入时支持自动补全，会显示已有的匹配标签供选择
- 系统预设了以下标签分类供参考：
  - **动物** — 猫、狗、鸟、鱼等
  - **自然** — 山水、森林、海洋、天空等
  - **风格** — 写实、卡通、油画、水彩等
  - **人物** — 男性、女性、儿童、老人等
  - **物品** — 建筑、车辆、食物、工具等
  - **色彩** — 暖色调、冷色调、单色、多彩等
  - **构图** — 特写、全景、对称、黄金比例等

### 4.4 归档图片

归档功能将图片从待整理移动到正式的归档目录，并按照命名模板重命名文件。

**单张归档：**
1. 点击图片卡片打开编辑弹窗
2. 完善图片信息后，点击"保存并归档"按钮

**批量归档：**
1. 使用 Ctrl+A 全选或按住 Ctrl 逐个点击选择多张图片
2. 按下 `Ctrl+Enter` 执行批量归档

归档后，图片文件会按照设置页面中配置的命名模板进行重命名，并移动到对应的归档目录中。

### 4.5 水印管理

- **手动检测：** 在编辑弹窗中可以手动触发水印检测
- **水印去除：** 使用边框采样 + 渐变混合 + 模糊平滑算法去除水印，尽可能保持图片画质
- **批量操作：** 支持对多张图片同时执行水印检测和去除

### 4.6 文件夹扫描与清理

通过文件夹扫描功能，可以实现新图片的手动导入和旧记录的手动清理：

1. 打开设置页面（`Ctrl+,`）
2. 切换到相应的标签页
3. 点击“扫描”按钮，根据结果选择“导入”或“清理”

根据扫描结果，系统会提示发现未入库的图片或已在硬盘上失效的数据库记录。你可以一键将未入库图片批量导入，或清理无效的数据库记录。

### 4.7 暗色模式

系统支持三种主题模式，可通过顶部栏的主题切换按钮进行切换：

- **浅色模式** — 点击太阳图标
- **深色模式** — 点击月亮图标
- **跟随系统** — 点击显示器图标，自动匹配操作系统的主题设置

主题选择会自动保存，下次启动应用时会恢复上次使用的主题。
你也可以在设置页的“通用设置”中分别配置普通模式和暗黑模式的主题色。

### 4.8 设置页面

按 `Ctrl+,` 或点击顶部栏的设置图标打开设置页面。设置页面包含五个标签页：

**通用设置：**
- 主题色 — 可分别设置普通模式和暗黑模式的主色
- 命名模板 — 配置归档时的文件命名规则
- 自定义待整理路径 — 设置导入图片时的临时存储目录
- 自定义归档路径 — 设置图片归档的目标目录
- 保存设置 — 保存成功后会自动关闭设置弹窗

**水印设置：**
- 自动检测开关 — 控制导入时是否自动检测水印
- 置信度阈值 — 调整水印检测的灵敏度
**扫描设置：**
- 扫描待整理目录 — 手动扫描并导入遗漏文件或清理失效记录
- 扫描归档目录 — 扫描归档文件夹，自动重命名并入库

**厂商管理：**
- 厂商和模型管理 — 添加、编辑、删除厂商及模型

**快捷键：**
- 快捷键列表 — 查看所有可用的键盘快捷键

### 4.9 云端双向同步 (全新 ☁️)

系统内置了工业级的全量双向云端同步能力，确保你在任何设备上都能无缝衔接工作：

**配置同步：**
1. 打开设置页面（`Ctrl+,`），切换到 **“同步与云端”** 标签页。
2. 输入你的私有 `服务端 URL` (例如：`https://sync.yourdomain.com`) 以及专属的 `API Key`。
3. 点击“保存并测试”，系统会验证服务器连通性。如果连通，上方工具栏的云朵图标将被激活。

**进行同步：**
1. 切换到 `sanIP` 界面。
2. 点击上方工具栏的 **【同步】** 按钮。
3. 同步过程是**增量且无感的**，它只上传你修改过的内容。
4. 成功后会弹出绿色提示框，精准报告“推送/拉取了多少条”（点击甚至能展开查看新增、修改、删除的具体明细！）。

*注：服务端采用类似 Git 的 CAS (内容寻址存储) 架构。同一张图片传一百次也只占用一份物理空间，极大地节省了云端存储！*

## 六、数据存储

- **数据库位置：** `%USERPROFILE%/.ai-image-manager/database.sqlite`
- 数据库中存储了所有图片的元数据、标签信息、厂商配置、应用设置等
- 图片文件本身保持在原始导入位置，不会复制或移动到数据库目录中
- 如需备份数据，只需复制上述数据库文件即可

## 七、常见问题

### Q: 首次启动编译很慢？

A: Rust 首次编译需要下载并编译所有依赖（包括 Tauri 核心库、SQLite 绑定等），通常需要 3-5 分钟。后续启动会快很多，因为编译产物已被缓存。如果网络较慢，可以考虑配置 Rust 镜像源加速下载。

### Q: 编译报错 "cargo not found"？

A: 这说明 Rust 尚未安装或未正确配置到系统 PATH 中。请访问 [https://rustup.rs](https://rustup.rs) 下载并运行安装程序，安装完成后重启终端（或重启电脑）使环境变量生效。

### Q: Windows 上编译报错 LNK1104？

A: 这是 MSVC 链接器的临时文件权限问题。尝试设置自定义临时目录来解决：

```powershell
$env:TEMP = "C:\Users\你的用户名\rust-temp"
mkdir $env:TEMP -Force
```

将"你的用户名"替换为实际的用户名，然后重新运行构建命令。

### Q: 启动报错 "Failed to init database: 9 values for 10 columns"？

A: 这是数据库初始化的字段数量不匹配问题，已在最新代码中修复。请确保使用最新代码。models 表有 10 个字段，INSERT 语句需要包含所有字段值。如果问题仍然存在，可以删除数据库文件让系统重建（参见下方"如何重置数据库"）。

### Q: 虚拟机中无法启动？

A: 部分虚拟机环境（尤其是没有 GPU 加速的配置）不支持 WebView2 的图形渲染，可能导致应用窗口无法正常显示。建议在物理机上运行本应用。

### Q: 如何重置数据库？

A: 删除数据库文件即可。数据库位于 `%USERPROFILE%/.ai-image-manager/database.sqlite`，删除后重启应用，系统会自动重建一个全新的空数据库。**注意：此操作会丢失所有已保存的图片元数据和配置信息。**

### Q: 如何添加自定义厂商？

A: 目前自定义厂商需要通过数据库操作或 API 接口添加，暂未提供图形界面。后续版本将提供厂商管理 UI，届时可以直接在设置页面中添加、编辑和删除厂商信息。

## 八、技术支持

本项目的核心技术栈如下：

- **桌面框架：** Tauri 2.x — 基于 Rust 的轻量级桌面应用框架
- **前端框架：** React 18 + TypeScript
- **UI 组件库：** Tailwind CSS + shadcn/ui
- **状态管理：** Zustand
- **数据库：** SQLite（通过 rusqlite 绑定）
- **后端语言：** Rust

如遇到问题，请确认已使用最新代码并参考上述常见问题排查。若问题仍未解决，请提供详细的错误信息和运行环境（操作系统版本、Node.js 版本、Rust 版本）以便定位问题。
