# 浏览器功能更新说明

## 问题描述

在之前的版本中，应用使用 `<iframe>` 来实现内嵌浏览器功能。但是许多网站（特别是 Google Gemini、ChatGPT 等）会设置安全策略来阻止在 iframe 中加载，导致显示"拒绝连接"的错误。

### 技术原因

1. **X-Frame-Options**: 网站设置此 HTTP 头来防止被嵌入到 iframe 中，防止点击劫持攻击
2. **Content-Security-Policy**: 现代网站使用 CSP 策略限制可以嵌入它们的来源
3. **Same-Origin Policy**: 浏览器的同源策略限制了 iframe 的功能

## 解决方案

将内嵌浏览器改为"快速启动器"模式，点击按钮后在用户的默认浏览器中打开网站。

### 优势

1. **完全兼容**: 所有网站都可以正常访问，没有安全限制
2. **更好的性能**: 使用系统浏览器，性能更好
3. **完整功能**: 用户可以使用浏览器的所有功能（扩展、书签等）
4. **更安全**: 避免在应用内处理敏感的登录信息

### 工作流程

1. 在应用的"浏览器"标签页中点击预设网站或输入自定义 URL
2. 网站在默认浏览器中打开
3. 在浏览器中生成 AI 图片
4. 下载图片到本地
5. 将图片拖拽到应用的"转换盒"标签页中导入

## 功能特性

### 预设网站

- **Gemini**: Google 的 AI 对话和图片生成工具
- **DALL-E**: OpenAI 的 ChatGPT 和 DALL-E 图片生成
- **Midjourney**: 专业的 AI 艺术图片生成平台

### 自定义 URL

用户可以输入任何网址，应用会自动添加 `https://` 协议（如果缺失）。

### 用户提示

应用会显示清晰的说明，告知用户：
- 为什么无法使用内嵌浏览器
- 如何使用外部浏览器
- 如何将生成的图片导入到应用中

## 技术实现

### 前端变更

**文件**: `src/components/WebBrowserView.tsx`

主要变更：
1. 移除 `<iframe>` 实现
2. 使用 `@tauri-apps/plugin-shell` 的 `open()` API 打开外部浏览器
3. 添加说明卡片，解释功能变更
4. 改进 UI，使用卡片式布局展示预设网站
5. 添加 toast 通知，提供操作反馈

### 依赖

- `@tauri-apps/plugin-shell`: Tauri 的 shell 插件，用于打开外部 URL
- 已在项目中配置，无需额外安装

## 用户体验改进

1. **清晰的说明**: 黄色提示卡片解释为什么使用外部浏览器
2. **使用提示**: 告知用户如何将图片导入应用
3. **美观的界面**: 使用卡片式布局，每个网站都有图标和描述
4. **操作反馈**: 使用 toast 通知告知用户操作结果
5. **键盘支持**: 在输入框中按 Enter 键可以快速打开网站

## 未来可能的改进

如果需要真正的内嵌浏览器功能，可以考虑：

1. **Tauri WebView 窗口**: 创建独立的 WebView 窗口（而不是 iframe）
   - 需要添加 Rust 代码来管理窗口
   - 可以避免 iframe 的限制
   - 但仍可能受到某些网站的限制

2. **浏览器扩展**: 开发浏览器扩展，直接从浏览器导入图片到应用
   - 需要额外的开发工作
   - 需要用户安装扩展

3. **API 集成**: 直接集成 AI 图片生成服务的 API
   - 需要 API 密钥
   - 可能需要付费
   - 但可以提供最流畅的体验
