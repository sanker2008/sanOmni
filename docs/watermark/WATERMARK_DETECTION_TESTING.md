# 水印检测功能测试指南

## 改进总结

### 主要改进

1. **多策略融合检测**
   - Gemini: 5 种检测策略（边缘、文字行、字符模式、颜色、阴影）
   - DALL-E: 4 种检测策略（颜色多样性、点阵模式、高频内容、饱和度）
   - Midjourney: 4 种检测策略（文字行、小文字、文字颜色、字符模式）

2. **降低检测阈值**
   - 从 0.5 降低到 0.4
   - 提高检测率，减少漏检

3. **自适应区域大小**
   - 根据图片尺寸动态调整检测区域
   - 设置最小/最大值防止极端情况

4. **添加 Toast 通知**
   - 检测完成后显示结果
   - 清晰的成功/失败反馈

## 测试步骤

### 1. 启动开发服务器

```bash
pnpm run tauri:dev
```

### 2. 准备测试图片

准备以下类型的图片：

#### Gemini 图片
- 右下角有 "AI-generated" 或类似文字水印
- 不同背景颜色（浅色、深色）
- 不同分辨率（512x512, 1024x1024, 2048x2048）

#### DALL-E 图片
- 右下角有彩色点阵水印
- DALL-E 3 生成的图片

#### Midjourney 图片
- 右下角有用户名和版本信息
- 不同版本（V5, V6, Niji）

#### 无水印图片
- 普通照片
- 其他 AI 工具生成的图片（Stable Diffusion, Leonardo AI 等）

## 评估指标

### 1. 准确率 (Accuracy)

```
准确率 = (正确检测数 / 总测试数) × 100%
```

目标：> 80%

### 2. 召回率 (Recall)

```
召回率 = (检测到的水印数 / 实际有水印数) × 100%
```

目标：> 75%（尽量不漏检）

### 3. 精确率 (Precision)

```
精确率 = (正确检测的水印数 / 报告有水印数) × 100%
```

目标：> 85%（尽量不误报）

### 4. F1 分数

```
F1 = 2 × (精确率 × 召回率) / (精确率 + 召回率)
```

目标：> 0.80

## 常见问题

### Q1: 检测不到 Gemini 水印

**可能原因：**
- 水印文字太小或太淡
- 背景颜色与文字颜色对比度低
- 水印位置不在右下角

**解决方案：**
- 调整检测区域大小
- 降低阈值
- 增加对比度增强预处理

### Q2: 误报其他内容为水印

**可能原因：**
- 图片右下角有类似文字的内容
- 检测阈值过低

**解决方案：**
- 提高置信度阈值
- 增加更多特征检测
- 添加 OCR 验证

### Q3: DALL-E 点阵检测不准

**可能原因：**
- 点阵图案变化
- 颜色组合不同

**解决方案：**
- 收集更多样本
- 调整颜色多样性阈值
- 使用模板匹配

### Q4: Midjourney 检测失败

**可能原因：**
- 文字太小
- 字体变化
- 背景复杂

**解决方案：**
- 增加小文字检测灵敏度
- 使用 OCR 识别
- 添加模板匹配

## 调优建议

### 如果召回率低（漏检多）

1. **降低阈值**
   ```rust
   has_watermark: confidence > 0.3  // 从 0.4 降低到 0.3
   ```

2. **调整权重**
   - 提高最有效策略的权重
   - 降低不稳定策略的权重

3. **扩大检测区域**
   ```rust
   let region_width = (width as f32 * 0.15).max(150.0).min(300.0) as u32;
   ```

### 如果精确率低（误报多）

1. **提高阈值**
   ```rust
   has_watermark: confidence > 0.5  // 从 0.4 提高到 0.5
   ```

2. **添加更多验证**
   - 增加新的检测策略
   - 要求多个策略同时满足

3. **缩小检测区域**
   ```rust
   let region_width = (width as f32 * 0.10).max(100.0).min(200.0) as u32;
   ```
