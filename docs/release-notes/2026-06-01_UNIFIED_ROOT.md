# 更新日志：统一根目录功能 (2026-06-01)

## 🎉 新增特性

### 统一根目录 (Unified Root Directory)

为了解决用户需要分别配置多个子目录（如待整理路径、归档路径、实验室目录等）的繁琐问题，我们在应用中引入了**“统一根目录”**功能。

#### 核心优势
1. **一键全局配置**：在“设置 -> 通用设置”中配置一次统一根目录，系统内的图片资产、IP 数据、实验室草稿等将会自动迁移存储至该目录下的对应子文件夹。
2. **架构安全解耦**：系统核心配置与数据库 (`database.sqlite`) 仍保留在系统默认的 AppData 目录中，确保系统的稳定启动和数据安全；而占用大量磁盘空间的多媒体数据则可以随统一根目录轻松转移至其他磁盘或 NAS 中。
3. **优先级兼容**：系统依然保留了针对各个细分功能的“自定义路径”设置。如果在某个特定模块（如 `IP 归档路径` 或新增的 `自定义作品集根目录`）设置了独立路径，该路径的优先级将高于“统一根目录”，满足高级用户的定制需求。

### 自定义作品集根目录 (Works Collection Path)
- 在“IP 设置”中新增了**自定义作品集根目录**的配置卡片，补全了作品集（Works）独立管理的能力。现在“作品集及衍生角色”的资产既可以完美地跟随统一根目录进行无缝迁移，也可以单独配置到其他磁盘独立存放。

## 🔧 技术实现

- 前端：新增了 `getAppRoot()` 工具函数接管了所有底层对 Tauri `appDataDir()` 的直接调用，动态读取统一根目录配置并实现 UI 的无缝预览展示。在 IP 设置中开放了作品集专属独立入口（`customWorksPath`）。
- 后端：在 Rust 命令中实现了基于 SQLite 配置表的动态路径解析器 `get_app_root` 和 `get_works_base_path`，深度替换了硬编码的数据落盘路径逻辑。

## 🐛 Bug 修复

- 修复了版本检查更新失败（404）时，错误提示不够友好的问题。
- **配置持久化问题**：修复了底层工具函数读取错误 localStorage 键名的问题（从 `san_settings` 更正为 `ai-image-manager-settings`），确保全局的统一根目录设置对所有依赖路径的模块生效。
- **Labs 工具兼容性**：
  - 重构了所有实验小工具（`ai-image-editor`, `image-compressor`, `image-slicer`, `product-image-maker`）的路径获取逻辑，统一调用 `getLabsRoot()`，使其完全支持自定义的 Labs 目录以及全局统一根目录。
  - 修复了部分实验小工具（特别是商品图制作模块）硬编码使用 `explorer` 打开目录导致在 Mac 环境下出现 `os error 2` 的问题，现已全部更换为跨平台动态检测执行 `open`（Mac）或 `explorer`（Windows）的方案。
