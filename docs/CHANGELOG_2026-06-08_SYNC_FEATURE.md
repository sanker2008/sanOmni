# 更新日志：云端双向同步与独立服务端 (2026-06-08)

经过对底层架构的重大升级，我们正式推出了**全量双向云端同步功能**，并为其配套开发了专属的高性能独立服务端 (`sanomni-sync-server`)。这项功能将彻底打破设备孤岛，让你在多台电脑间无缝流转灵感与资产。

## 🌟 核心亮点

### 1. 工业级同步引擎
- **无感增量同步**：底层 SQLite 数据库现已全面接入专属触发器（Triggers）。你在客户端的任何增删改操作（哪怕只是修改一个标签），都会被瞬间捕获并记录在底层的 Changelog 中。点击同步时，只传输有差异的数据，极大地节省了时间和带宽。服务端采用原生的 `INSERT OR REPLACE` 处理，完美避免主键冲突。
- **详尽明细反馈**：同步成功后，UI 界面不仅会提示同步完成，更会智能展现具体的变更明细（例如：新增了多少条记录、修改了多少、删除了多少），让你对数据流向了如指掌。
- **云端历史与强制重推**：在设置页中提供了**实时的服务端同步日志面板**，每一条数据的变动版本均可追溯。针对状态不一致或数据遗漏的极端情况，新增了**“强制全量重推”**功能，可一键将本地全量数据安全补齐至云端，且不会造成物理文件重复传输。

### 2. Git 级 CAS 内容寻址存储
服务端并没有采用传统的按文件名存储的方式，而是引入了与 Git 底层一致的 **CAS（Content-Addressable Storage，内容寻址存储）架构**：
- **物理去重**：每张图片上传时都会通过 SHA-256 算法计算出全网唯一的哈希值。即便你在不同项目中反复使用同一张素材，服务端也只保存一份物理文件，将存储效率压榨到极致。
- **孤立文件垃圾回收保护**：当你在本地删除图片时，系统只会优雅地切断引用记录，而不会立刻贸然删除服务器上的物理原图，彻底杜绝了因多设备引用冲突导致的文件丢失。

### 3. 高安全与强鲁棒性
- **系统级密钥链保护**：你的私有 API Key 将被安全地存放在操作系统的底层 Keyring（凭据管理器）中，前端界面自动按需调取，拒绝明文泄露。
- **大体积负载应对**：服务端框架（Axum）已全面升级限流策略，单次允许最高 **100MB** 的巨型变更包传输，完美应对初次海量导入和深度批量修改的极端场景，彻底告别 `error sending request` 等断流问题。

## 🛠️ 如何配置与使用

1. **部署服务端**：请参考 `sanomni-sync-server` 目录下的 `README.md` 在你自己的云服务器（或 NAS 等）上搭建。
2. **连接客户端**：在 sanOmni 的设置面板（`Ctrl+,`）中找到 **同步与云端**，填入你的服务端 URL 及密码。
3. **一键同步**：切换到 `sanIP` 界面，点击顶部工具栏的 **刷新/同步** 按钮，尽情享受丝滑的数据穿梭体验吧！
