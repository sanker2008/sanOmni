# 更新日志：统一根目录功能 (2026-06-01)

## 🎉 新增特性

### 统一根目录 (Unified Root Directory)

为了解决用户需要分别配置多个子目录（如待整理路径、归档路径、实验室目录等）的繁琐问题，我们在应用中引入了**“统一根目录”**功能。

#### 核心优势
1. **一键全局配置**：在“设置 -> 通用设置”中配置一次统一根目录，系统内的图片资产、IP 数据、实验室草稿等将会自动迁移存储至该目录下的对应子文件夹。
2. **架构安全解耦**：系统核心配置与数据库 (`database.sqlite`) 仍保留在系统默认的 AppData 目录中，确保系统的稳定启动和数据安全；而占用大量磁盘空间的多媒体数据则可以随统一根目录轻松转移至其他磁盘或 NAS 中。
3. **优先级兼容**：系统依然保留了针对各个细分功能的“自定义路径”设置。如果在某个特定模块（如 `IP 归档路径`）设置了独立路径，该路径的优先级将高于“统一根目录”，满足高级用户的定制需求。

## 🔧 技术实现

- 前端：新增了 `getAppRoot()` 工具函数接管了所有底层对 Tauri `appDataDir()` 的直接调用，动态读取统一根目录配置并实现 UI 的无缝预览展示。
- 后端：在 Rust 命令中实现了基于 SQLite 配置表的动态路径解析器 `get_app_root`，深度替换了硬编码的数据落盘路径逻辑。
