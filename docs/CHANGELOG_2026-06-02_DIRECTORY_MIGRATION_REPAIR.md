# 更新日志 - 目录迁移与数据修复 (2026-06-02)

本次更新核心聚焦于应用底层“目录路径管理”和“文件资产安全性”的大幅强化，提供了一套能够抵抗各类异常场景的健壮数据维护机制，同时彻底改造了相关的 UI 交互体验。

## 🌟 核心新功能 (New Features)

### 1. 安全的全量数据迁移
当在“设置”中修改任何关联了真实文件的目录配置（如统一根目录、待整理目录、归档目录等）时，系统现在具备了极高的转移安全性：
- **跨盘符支持与降级机制**：在跨盘符（如 C盘 移动至 D盘）无法直接重命名目录时，系统会自动降级执行底层深拷贝（Deep Copy）+ 验证 + 原路删除的机制，确保任何情况下都不会发生丢数据的情况。
- **关联数据库自重构**：单纯移动文件并不能让原有的图片在应用中显示。本次更新深度接入了 SQLite 事务管理（Transaction），系统会在文件移动完成后，对底层核心库（涵盖 `images`、`ip_assets`、`works`、`ip_sticker_packs` 等所有含绝对路径字段的表）执行精确的路径前缀替换。

### 2. 交互升级：自定义 React Web 弹窗
- **移除简陋原生弹窗**：抛弃了原先通过 Tauri 调用的操作系统默认 Confirm 弹窗，引入了完全由 React + shadcn 驱动的高颜值 Dialog 面板。
- **多目录变更聚合展示**：支持一次性保存多个修改的目录设置，并以清晰的 `旧路径 ➔ 新路径` 的合并视图进行集中提示，提供更加人性化的 `仅保存` 和 `保存并迁移` 的多选按钮组合。

### 3. 数据诊断与一键修复模块
这是位于“通用设置”底部的全新工具，专为处理不可抗力因素（如用户手动在资源管理器中把文件夹移走了，或者重装了系统）导致的图片路径失效（断链）而生。
- **全量扫描与统计**：一键检测当前数据库中所有历史数据在实际硬盘上的状态，统计出“正常”、“失效”以及“可修复”的条目。
- **一键自愈 (Auto-Fix)**：系统会利用智能算法，在当前配置的合法根目录以及所有归档目录内，通过文件名指纹尝试重新匹配丢失的文件。一旦匹配成功，即刻重连数据库记录。

## 🐞 修复与优化 (Bug Fixes & Improvements)

- 🐛 **修复 Loading 死锁问题**：加入了严格的 `finally` 生命周期回收机制，无论跨盘迁移过程中遭遇任何系统级（权限不足、文件占用）错误，都能确保安全解锁 UI。
- ⚡ **缓存同步强刷策略**：在发生大规模数据迁移或修复完成后，通过生命周期级的 `window.location.reload()` 清洗陈旧缓存，完美避免了任何可能的 React 状态残留 Bug。
- 🎨 **体验微调**：为复杂的设置项补充了友好的中文提示文案；弹窗的布局逻辑更贴近现代 App 交互标准。
