# 高级抠图 (Pro) 算法与原理说明

本文档主要记录 `sanLabs` 工具箱中“高级抠图 (Pro)” (`pro-background-removal`) 小工具的核心算法与实现原理，以便后期调整与借鉴。

该工具旨在提供两种不同的策略以应对不同的场景：
1. **策略A (表情包/纯色文字)**：利用 `pillow_matting.py`，针对白底、绿幕、蓝幕等具有纯色背景的图片进行极速抠图与边缘羽化处理。
2. **策略B (复杂人像/发丝)**：利用深度学习模型 (IS-Net 等) 处理真实世界中的复杂人像和细微毛发。

以下重点阐述**策略A（传统图像处理策略）**的核心革新算法。

## 1. 痛点：传统绝对距离算法的缺陷

早期的抠图算法使用 RGB 的“绝对距离” (Chebyshev 距离) 来判断像素是否属于背景：
```python
dist = max(|R - R_bg|, |G - G_bg|, |B - B_bg|)
closeness = 255 - dist
```

**缺陷分析：**
在图像的抗锯齿（羽化）边缘处，前景色和背景色会发生物理混合。例如：白色衣服 `(255, 255, 255)` 和绿色背景 `(0, 255, 0)` 混合出的边缘像素是浅绿色 `(127, 255, 127)`。
- 这个浅绿色的 `dist = 127`，距离纯绿色依然非常远。
- 导致算法判定该像素为“实体前景”，如果不把透明阈值拉得极低（这会导致主体也变透明），这些边缘像素将始终保持 100% 不透明。
- 最终结果：**在抠出的主体周围残留一圈非常明显的绿边/白边。**

## 2. 核心算法一：基于“色彩纯度”的 Alpha 遮罩引擎

为了解决上述问题，对于带色彩的背景（如绿幕、蓝幕），我们将底层逻辑重构为基于“色彩纯度 (Diff)”的智能计算：

**算法逻辑：**
```python
# 找出背景色的主要通道 (dom_idx)
# 对于绿幕：
diff = G - max(R, B)
```
**原理说明：**
- `diff` 直接衡量了像素中“目标颜色突出的程度”。
- 哪怕是 50% 混合的浅绿色 `(127, 255, 127)`，它的 `diff = 255 - 127 = 128`。这个数值明确指出它带有绿光污染。
- 相比绝对距离算法，**纯度算法能极度敏锐地识别出所有被背景污染的过渡边缘**，并赋予它们正确的半透明度（Alpha），而非一刀切判定为实体。

**UI 滑块映射：**
为了确保前端的默认“不透明阈值”和“全透明阈值”滑块 (通常为 220 ~ 250) 能够平滑控制新的 `diff` 算法，我们在底层做了一个线性映射：
```python
closeness = diff + 215
```
这样，最完美的边缘阈值 (如 5 到 35 的纯度) 就刚好映射到了 220 到 250 的滑块范围内。

## 3. 核心算法二：完全解耦的溢色压制 (Spill Suppression)

仅仅赋予正确的透明度是不够的。半透明的绿边贴在白色画布上，依然会显示出“半透明的绿色”。必须将物理混入的绿色彻底抽离。

**痛点：**
如果用户在前端强制将阈值拉到极限（例如下限 255，上限 135），会导致所有的像素强制变成 `Alpha = 1.0` (100% 不透明)。如果去色算法依赖 `Alpha < 1.0` 才能触发，就会导致去绿边功能完全失效。

**创新解法：解耦去色**
将色彩剥离引擎与用户的 UI 透明度设置**完全解耦**。无论像素最终的 Alpha 值是多少，只要它的色彩纯度 `diff` 达标，就无条件强制执行去色。

**算法逻辑（以绿幕为例）：**
```python
# 根据纯度计算出一个溢色系数 spill_amount (范围 0.0 到 1.0)
spill_amount = np.clip((diff - 5) / 30.0, 0.0, 1.0)

# 强制压制绿色通道：绿色通道绝不能超过红蓝通道的最高值
suppressed_G = min(G, max(R, B))

# 混合去色：受污染越严重的像素，去色力度越强
final_G = G * (1.0 - spill_amount) + suppressed_G * spill_amount
```

**效果：**
无论用户的透明度滑块怎么拉，边缘只要泛着绿光，绿色通道就会被这套引擎强制掐断并替换为灰白中性色。这就是为什么抠出来的图能够达到极致锐利且毫无绿边（拔毛级）的根本原因。

## 4. 其它场景回退机制

- **白底/灰底/黑底商品图**：此时背景没有明显的色彩主导（即红绿蓝三色差值不大）。
- 程序会自动回退到极简模型：
  - 白底：`closeness = min(R, G, B)` （越暗的颜色越被视为前景）。
  - 黑底：`closeness = 255 - max(R, G, B)` （越亮的颜色越被视为前景）。
