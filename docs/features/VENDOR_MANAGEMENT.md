# 厂商管理功能

## 概述

现在可以在设置页面中管理所有 AI 图片生成厂商和对应的模型。

## 功能特性

### 1. 显示所有厂商

- **已归档视图**：左侧厂商分类现在显示所有厂商，包括没有图片的厂商
- 每个厂商旁边显示图片数量
- 可以展开/折叠查看厂商下的模型

### 2. 厂商管理（设置 → 厂商管理）

#### 查看厂商列表
- 显示所有厂商及其模型
- 显示每个厂商下的模型数量
- 可展开/折叠查看模型详情

#### 添加厂商
1. 点击"添加新厂商"按钮
2. 输入厂商名称（如：OpenAI）
3. 输入路径标识（如：openai，用于文件路径）
4. 点击保存图标确认

#### 编辑厂商
1. 点击厂商右侧的编辑图标
2. 修改厂商名称或路径标识
3. 点击保存图标确认
4. 点击 X 图标取消编辑

#### 删除厂商
1. 点击厂商右侧的删除图标
2. 确认删除操作
3. 厂商将被软删除（is_active = 0）

### 3. 模型管理

#### 添加模型
1. 展开对应的厂商
2. 点击"添加模型"按钮
3. 输入模型名称（如：GPT Image 2）
4. 输入路径标识（如：gpt-image-2）
5. 可选：输入描述信息
6. 点击保存图标确认

#### 编辑模型
1. 点击模型右侧的编辑图标
2. 修改模型信息
3. 点击保存图标确认
4. 点击 X 图标取消编辑

#### 删除模型
1. 点击模型右侧的删除图标
2. 确认删除操作
3. 模型将被软删除（is_active = 0）

## 技术实现

## 数据库说明

### 软删除机制

厂商和模型使用软删除机制：
- 删除时不会真正从数据库中移除记录
- 而是将 `is_active` 字段设置为 0
- 这样可以保留历史数据，避免破坏已有图片的关联关系

### 排序规则

厂商和模型按以下顺序排序：
1. `sort_order` 字段（升序）
2. `name` 字段（升序）

## 使用建议

1. **路径标识命名规范**
   - 使用小写字母
   - 使用连字符分隔单词
   - 避免使用空格和特殊字符
   - 示例：`openai`, `google-imagen`, `midjourney`

2. **厂商和模型的关系**
   - 每个模型必须属于一个厂商
   - 删除厂商前建议先删除或迁移其下的模型
   - 模型的 ID 会自动生成为 `{vendor_id}-{model_path}`

3. **数据一致性**
   - 修改厂商或模型的路径标识不会影响已有图片
   - 已归档的图片会继续显示原有的厂商和模型信息
   - 建议在有图片关联前完成厂商和模型的配置

## 快捷键

- `Ctrl + ,` - 打开设置
- 在设置中点击"厂商管理"标签页即可管理厂商

## 注意事项

1. 删除操作是软删除，不会真正删除数据
2. 修改路径标识不会影响已有图片的关联
3. 建议在导入图片前配置好厂商和模型
4. 厂商和模型的名称支持中文和英文
