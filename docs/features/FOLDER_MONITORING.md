# 文件夹扫描与清理功能说明

## 功能概述

**文件夹扫描**功能允许你主动扫描“待整理”（Inbox）和“已归档”（Archived）目录，以识别出未被系统数据库记录的新图片，或者清理出已经被你从硬盘上手动删除但依然存在于数据库中的“幽灵”记录。

出于对用户数据控制权和避免错误导入不必要图片的考虑，我们移除了原有的后台“文件夹自动监控”功能。现在所有的扫描和清理操作都需要**用户手动触发并确认**，确保你的灵感库和资产库的绝对纯净。

## 使用场景

### 场景1：发现新下载的图片
如果你通过外部工具（如浏览器下载、云盘同步）直接将图片保存在了你的“待整理”目录下，你可以通过“扫描”功能一次性将它们导入数据库。

### 场景2：清理硬盘空间
如果你使用系统的资源管理器，直接在硬盘的“待整理”目录里删除了大量的废弃图片。由于你绕过了软件，软件的数据库中仍会有这些图片的记录。此时，你可以运行扫描并点击“确认清理”，一键清理掉这些已经不存在于硬盘上的无效记录。

## 使用方法（待整理区域）

### 1. 扫描待整理目录
- 打开“设置”面板（快捷键 `Ctrl + ,`）。
- 切换到 **sanPrompt** 或 **sanIP** 标签页（根据你需要扫描的类型决定）。
- 找到“扫描待整理目录”卡片，点击 **开始扫描**。

### 2. 查看与确认扫描结果
扫描完成后，下方会展示本次扫描的结果面板。包含两项关键数据：
- **新增文件 (未入库)**：这代表硬盘上存在，但软件数据库中没有记录的图片。
- **失效记录 (文件已删除)**：这代表软件数据库中有记录，但你已经在硬盘上将该文件删除了的记录。

### 3. 执行操作
根据扫描结果，系统会提供相应的操作按钮，**完全由你决定是否执行**：
- **导入文件**：如果你确认这些“新增文件”是你需要的，点击此按钮，系统会将它们批量导入待整理。
- **确认清理**：如果你确认那些“失效记录”是你已经不要的，点击此按钮，系统会将它们从数据库中清除。

## 使用方法（归档区域）

对于归档区域的扫描（例如你直接将旧的文件库拷贝到了 `archived` 目录下）：
- 在设置中找到“扫描归档目录”。
- 点击 **开始扫描**。
- 系统会遍历归档目录，根据你设置的命名模板重命名未入库的文件，并将它们直接登记到归档数据库中。

## ⚠️ 注意事项
- 所有扫描过程都不会擅自删除你的本地硬盘文件。
- 对于 `missing_on_disk` 的记录清理，本质上只是**从数据库中抹除**。
- 如果扫描发现了无法自动分类的“新增文件”，在“导入文件”后，它们可能会被标记为 `unknown` 的厂商或模型，你需要后续在应用内手动打标签或修改信息。
