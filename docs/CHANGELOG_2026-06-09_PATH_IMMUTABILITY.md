# 数据底层防断链与同步安全加固 (2026-06-09)

## 🌟 核心背景

在 sanOmni 系统架构中，许多核心业务实体（如 IP、厂商、AI 模型、作品、表情包等）都需要在本地文件系统中拥有自己专属的物理目录来存放关联的资源图片。

为了建立起数据库记录与物理文件的一一映射，系统引入了 `path` (路径标识) 字段。以往的设计允许在编辑这些实体时修改路径标识，但这会引发极其严重的潜在问题：

1. **本地文件断链**：仅仅修改了数据库里的 `path` 字段，已存在于旧路径（如 `ip_assets/{old_path}/...`）下的实体资源图片不会自动进行级联迁移，这会导致后续再也无法加载之前保存的图片资产。
2. **多设备云端同步错乱**：在加入了双向同步功能后，如果在设备 A 上更改了路径标识，而设备 B 尚未同步，随后在设备 A、B 上同时对该实体的不同图片进行操作，由于两端的底层路径发生分歧，这在云端同步对齐时将造成灾难性的路径冲突与文件丢失风险。

## ✨ 技术更新方案

为了彻底根除以上隐患，我们在架构层级与 UI 交互交互层级实施了**“路径硬锁定机制”**。

涉及的数据模型包括：
- `IpAsset` (IP 形象)
- `IpStickerPack` (IP 专属表情包套件)
- `Work` (影视/实体作品)
- `Vendor` (AI 生成厂商)
- `Model` (AI 生成模型)

**主要调整：**
- **创建时灵活配置**：所有的路径标识依旧允许在新建对应数据实体时自由配置或自动生成（要求依然是仅支持小写字母、数字及连字符组合）。
- **编辑时强制锁定**：一旦实体创建成功，底层文件系统目录框架即确立。此时在任何界面再次进入这些实体的“编辑模式”时，路径标识输入框会被灰态禁用（`disabled`）。
- **增加视觉警示**：在相关界面的路径输入框下方，加入了带有 `⚠️` 警告图标和醒目高亮颜色（橙黄色调）的提示文字：“为保证文件关联和同步安全，创建后路径标识不可修改”，帮助用户在第一次创建时就能做出明确的决定。

## 💡 业务价值

通过这一重构：
1. 本地磁盘资源目录与数据库记录之间的关联变得**绝对稳固且防篡改**。
2. 双向同步引擎不再需要额外处理极其复杂的跨目录、跨节点重命名数据迁移脏逻辑。
3. 用户层面通过防呆设计（Poka-yoke），能够避免因为无意间手滑修改名称而导致重要资产突然“消失”的系统级惊吓。
