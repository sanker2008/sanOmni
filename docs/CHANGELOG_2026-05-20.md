# 更新日志 - 2026-05-20

## 🎉 新增功能：厂商管理

### 概述

本次更新添加了完整的厂商和模型管理功能，用户现在可以在设置页面中自由管理所有 AI 图片生成厂商和对应的模型。

---

## ✨ 主要变更

### 1. 设置页面新增"厂商管理"标签

在设置页面中新增了"厂商管理"标签页，提供以下功能：

- **查看所有厂商**：显示系统中的所有厂商及其模型
- **添加厂商**：可以添加新的 AI 图片生成厂商
- **编辑厂商**：修改厂商的名称和路径标识
- **删除厂商**：软删除不需要的厂商
- **添加模型**：为厂商添加新的模型
- **编辑模型**：修改模型的名称、路径标识和描述
- **删除模型**：软删除不需要的模型

### 2. 已归档视图显示所有厂商

**之前**：左侧厂商分类只显示有图片的厂商

**现在**：显示所有厂商，包括没有图片的厂商，方便用户了解系统中配置的所有厂商

### 3. 软删除机制

厂商和模型的删除采用软删除机制：
- 删除时不会真正从数据库中移除记录
- 而是将 `is_active` 字段设置为 0
- 保留历史数据，避免破坏已有图片的关联关系

## 🔧 技术实现

### 前端更改

#### 1. SettingsView.tsx
- 添加"厂商管理"标签页
- 实现厂商和模型的 CRUD 界面
- 支持展开/折叠厂商查看模型
- 内联编辑模式，点击编辑图标即可修改

#### 2. ArchivedView.tsx
- 移除了 `if (count === 0) return null` 的过滤逻辑
- 现在显示所有厂商，即使没有图片

#### 3. tauri.ts
新增 API 方法：
- `vendorApi.update()` - 更新厂商
- `vendorApi.delete()` - 删除厂商
- `vendorApi.updateModel()` - 更新模型
- `vendorApi.deleteModel()` - 删除模型

### 后端更改

#### 1. vendors.rs
新增命令：
- `update_vendor()` - 更新厂商信息
- `delete_vendor()` - 软删除厂商
- `update_model()` - 更新模型信息
- `delete_model()` - 软删除模型

修改命令：
- `get_vendors()` - 移除 `WHERE is_active = 1` 过滤，显示所有厂商
- `fetch_models_by_vendor()` - 移除 `WHERE is_active = 1` 过滤，显示所有模型

#### 2. lib.rs
注册新的命令：
- `update_vendor`
- `delete_vendor`
- `update_model`
- `delete_model`

## 📖 使用指南

### 如何管理厂商

1. **打开设置**
   - 按 `Ctrl + ,` 快捷键
   - 或点击右上角的设置图标

2. **进入厂商管理**
   - 点击"厂商管理"标签页

3. **添加厂商**
   - 点击"添加新厂商"按钮
   - 输入厂商名称（如：OpenAI）
   - 输入路径标识（如：openai）
   - 点击保存图标

4. **编辑厂商**
   - 点击厂商右侧的编辑图标
   - 修改信息后点击保存图标

5. **删除厂商**
   - 点击厂商右侧的删除图标
   - 确认删除操作

### 如何管理模型

1. **展开厂商**
   - 点击厂商左侧的展开图标

2. **添加模型**
   - 点击"添加模型"按钮
   - 输入模型名称和路径标识
   - 点击保存图标

3. **编辑/删除模型**
   - 操作方式与厂商相同

## 💡 使用建议

### 路径标识命名规范

- 使用小写字母
- 使用连字符分隔单词
- 避免使用空格和特殊字符
- 示例：`openai`, `google-imagen`, `midjourney`

### 数据一致性

- 修改厂商或模型的路径标识不会影响已有图片
- 已归档的图片会继续显示原有的厂商和模型信息
- 建议在有图片关联前完成厂商和模型的配置
