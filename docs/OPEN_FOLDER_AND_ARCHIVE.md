# 打开文件夹和单个归档功能

## 新增功能

### 1. 打开图片所在文件夹

#### 功能说明
可以快速打开图片在文件系统中的实际位置，方便查看、管理或备份原始文件。

#### 使用方法

**方法1：右键菜单**
- 在图片卡片上点击鼠标右键
- 自动打开文件管理器并选中该图片

**方法2：悬停菜单**
- 将鼠标悬停在图片卡片上
- 点击右下角的文件夹图标 📁
- 打开文件管理器并选中该图片

#### 跨平台支持
- **Windows**: 使用 `explorer /select,` 打开资源管理器并选中文件
- **macOS**: 使用 `open -R` 打开 Finder 并选中文件
- **Linux**: 使用 `xdg-open` 打开文件管理器（打开文件夹）

#### 使用场景
- 查看图片的原始位置
- 快速访问同一文件夹中的其他文件
- 备份或移动原始文件
- 检查文件属性和元数据
- 使用其他工具编辑图片

### 2. 单个图片归档

#### 功能说明
可以直接归档单张图片，无需先选中再批量归档。

#### 使用方法
1. 将鼠标悬停在图片卡片上
2. 点击右下角的归档图标 📦
3. 图片会立即归档到对应的厂商/模型目录

#### 前提条件
图片必须满足以下条件才能归档：
- ✅ 已标记厂商（Vendor）
- ✅ 已标记模型（Model）

如果缺少这些信息，会提示：
```
图片已归档或缺少信息（厂商/模型）
```

#### 归档流程
1. 检查图片是否已标记厂商和模型
2. 从 inbox 移动文件到 archived 目录
3. 按照 `厂商/模型` 的层级结构组织
4. 根据命名模板重命名文件
5. 更新数据库记录
6. 从待整理界面移除
7. 显示归档结果提示

#### 归档位置
```
默认位置:
%APPDATA%\com.sanomni.app\archived\<厂商>\<模型>\

自定义位置:
<自定义归档路径>\archived\<厂商>\<模型>\
```

#### 命名模板
默认模板：`{vendor}-{model}-{date}-{index}`

示例：
```
midjourney-v6-2024-05-20-001.png
stable-diffusion-sdxl-2024-05-20-002.jpg
```

可在设置中自定义命名模板。

## 注意事项

### 打开文件夹功能
1. **权限要求**
   - 需要 shell 执行权限
   - 已在 capabilities 中配置

2. **跨平台差异**
   - Windows: 会选中文件
   - macOS: 会选中文件
   - Linux: 只打开文件夹（不同发行版可能不同）

3. **文件不存在**
   - 如果文件已被移动或删除
   - 会显示错误提示
   - 建议先检查文件是否存在

### 单个归档功能
1. **必须先标记**
   - 归档前必须标记厂商和模型
   - 可以通过快速编辑添加

2. **文件移动**
   - 归档会移动文件（不是复制）
   - 从 inbox 移动到 archived
   - 原 inbox 位置的文件会被删除

3. **命名冲突**
   - 如果目标文件名已存在
   - 会自动递增索引号
   - 例如：001, 002, 003...

4. **归档失败**
   - 可能原因：
     - 缺少厂商/模型信息
     - 目标目录无写入权限
     - 磁盘空间不足
   - 会显示具体错误信息

## 常见问题

### Q: 右键没有反应？
**A:** 
1. 确保点击在图片卡片区域
2. 检查是否有其他右键菜单拦截
3. 尝试使用悬停菜单的文件夹图标

### Q: 打开的文件夹位置不对？
**A:** 
- 打开的是图片的实际存储位置
- 待整理图片：inbox 目录
- 归档图片：archived 目录
- 不是原始导入位置

### Q: 单个归档提示缺少信息？
**A:** 
1. 双击图片打开快速编辑
2. 选择厂商（Vendor）
3. 选择模型（Model）
4. 保存后再尝试归档

### Q: 归档后图片在哪里？
**A:** 
- 默认位置：`%APPDATA%\com.sanomni.app\archived\<厂商>\<模型>\`
- 自定义位置：在设置中查看
- 可以在"已归档"标签页查看

### Q: 可以撤销归档吗？
**A:** 
- 目前不支持直接撤销
- 可以手动将文件移回 inbox 目录
- 或在"已归档"标签页中管理

### Q: Linux 下打开文件夹不选中文件？
**A:** 
- 这是 Linux 的限制
- 不同桌面环境行为不同
- 只能打开文件夹，需要手动找到文件

## 技术实现

### 打开文件夹
```typescript
const handleOpenFolder = async (e: React.MouseEvent) => {
  e.stopPropagation();
  const { Command } = await import("@tauri-apps/plugin-shell");
  
  // 获取文件所在目录
  const dirPath = image.absolute_path.substring(0, Math.max(
    image.absolute_path.lastIndexOf('/'),
    image.absolute_path.lastIndexOf('\\')
  ));

  // 根据操作系统打开文件管理器
  if (navigator.platform.toLowerCase().includes('win')) {
    await Command.create('explorer', ['/select,', image.absolute_path]).execute();
  } else if (navigator.platform.toLowerCase().includes('mac')) {
    await Command.create('open', ['-R', image.absolute_path]).execute();
  } else {
    await Command.create('xdg-open', [dirPath]).execute();
  }
};
```

### 单个归档
```typescript
const handleArchiveSingle = async (imageId: string) => {
  // 获取归档路径
  let libraryPath = settings.customArchivedPath || await appDataDir();
  
  // 调用归档 API
  const result = await imageApi.archive([imageId], libraryPath);
  
  // 处理结果
  if (result.success_count > 0) {
    removeImage(imageId);
    showMessage("成功归档 1 张图片");
  }
};
```
