# 水印检测算法改进

## 问题分析

原有的水印检测算法过于简单，主要依赖基础的像素分析，导致检测准确率低。

### 原算法的问题

1. **单一检测策略** - 只检查像素颜色和透明度
2. **阈值过于严格** - 置信度阈值 0.5 太高，容易漏检
3. **缺少文字特征检测** - 无法识别文字的边缘、笔画等特征
4. **没有多策略融合** - 单一特征容易误判

## 改进方案

### 新算法架构

采用**多策略融合**的方法，每个平台使用 4-5 种不同的检测策略，然后加权平均得到最终置信度。

```
检测结果 = Σ (策略i × 权重i)
```

### Gemini 水印检测

Gemini 水印特征：右下角灰白色文字 "AI-generated" 或类似文本

#### 检测策略

1. **文字边缘检测** (权重 25%)
   - 使用 Sobel 算子检测边缘
   - 文字区域应有 3-20% 的边缘像素
   - 方法：`detect_text_edges()`

2. **水平文字行检测** (权重 20%)
   - 分析每行的亮度分布
   - 寻找亮度峰值（文字行）和谷值（行间距）
   - Gemini 通常有 1-2 行文字
   - 方法：`detect_horizontal_text_lines()`

3. **字符模式检测** (权重 30%)
   - 检测垂直笔画（"AI"、"I"、"l" 等字符特征）
   - 统计连续的垂直亮像素
   - "AI-generated" 应有 3-15 个垂直笔画
   - 方法：`detect_gemini_text_pattern()`

4. **颜色一致性检测** (权重 15%)
   - Gemini 使用灰白色文字
   - 检查像素是否为灰度色（R≈G≈B）
   - 亮度在 150-240 之间
   - 方法：`detect_gemini_color_pattern()`

5. **阴影检测** (权重 10%)
   - Gemini 文字通常有微妙阴影
   - 检测亮像素后跟较暗像素的模式
   - 方法：`detect_text_shadow()`

#### 检测区域

- 位置：右下角
- 宽度：图片宽度的 12%，最小 120px，最大 250px
- 高度：图片高度的 4%，最小 20px，最大 60px

#### 置信度阈值

- **检测阈值**：0.4（降低以提高检测率）
- **高置信度**：> 0.7
- **中置信度**：0.4 - 0.7
- **低置信度**：< 0.4（不报告）

### DALL-E 水印检测

DALL-E 3 水印特征：右下角彩色点阵图案

#### 检测策略

1. **颜色多样性检测** (权重 30%)
   - DALL-E 使用多种颜色的点
   - 统计颜色种类数量
   - 计算颜色多样性分数
   - 方法：`detect_dalle_color_diversity()`

2. **点阵模式检测** (权重 35%)
   - 检测规则的点状图案
   - 采样网格，检查中心像素与邻居的差异
   - 点应该与周围像素有明显差异
   - 方法：`detect_dalle_dot_pattern()`

3. **高频内容检测** (权重 20%)
   - 点阵图案产生高频信号
   - 检测相邻像素的亮度变化
   - 高频比例应在 20-70%
   - 方法：`detect_high_frequency_pattern()`

4. **饱和度检测** (权重 15%)
   - DALL-E 的点颜色饱和度高
   - 统计高饱和度像素比例
   - 方法：`detect_high_saturation()`

#### 检测区域

- 位置：右下角
- 大小：图片短边的 8%，最小 60px，最大 150px（正方形）

#### 置信度阈值

- **检测阈值**：0.4
- **典型置信度**：0.6 - 0.9

### Midjourney 水印检测

Midjourney 水印特征：右下角小号白色文字（用户名、版本信息）

#### 检测策略

1. **文字行检测** (权重 30%)
   - 分析行亮度分布
   - 检测文字行高度（6-25 像素）
   - MJ 通常有 1-3 行文字
   - 方法：`detect_mj_text_lines()`

2. **小文字特征检测** (权重 25%)
   - 使用 flood fill 检测小像素簇
   - 小文字簇大小：5-60 像素
   - 应有 5-50 个小簇
   - 方法：`detect_small_text()`

3. **文字颜色检测** (权重 25%)
   - MJ 使用白色或浅色文字
   - 亮像素（>180）比例：8-40%
   - 方法：`detect_mj_text_color()`

4. **字符模式检测** (权重 20%)
   - 检测字符状区域（20-70% 亮像素）
   - 应有 3-20 个字符模式
   - 方法：`detect_mj_pattern()`

#### 检测区域

- 位置：右下角
- 宽度：图片宽度的 18%，最小 150px，最大 350px
- 高度：图片高度的 6%，最小 30px，最大 100px

#### 置信度阈值

- **检测阈值**：0.4
- **典型置信度**：0.5 - 0.8

## 算法优势

### 1. 多策略融合

- 单一策略失败不影响整体结果
- 多个策略相互验证，提高准确性
- 不同特征互补，覆盖更多情况

### 2. 自适应区域大小

- 根据图片尺寸动态调整检测区域
- 设置最小/最大值防止极端情况
- 适应不同分辨率的图片

### 3. 降低阈值

- 从 0.5 降低到 0.4
- 提高检测率，减少漏检
- 多策略融合保证准确性

### 4. 灰度图分析

- 对 Gemini 检测使用灰度图
- 减少颜色干扰
- 更好地检测文字边缘

### 5. 边缘检测

- 使用类 Sobel 算子
- 检测文字的边缘特征
- 比简单的亮度检测更准确

## 性能优化

### 1. 区域裁剪

- 只分析右下角小区域
- 避免全图扫描
- 大幅提升速度

### 2. 采样策略

- 某些检测使用网格采样
- 减少计算量
- 保持检测准确性

### 3. 早期退出

- 某些检测有大小限制
- 不符合条件直接返回 0
- 避免无效计算

## 测试建议

### 测试用例

1. **Gemini 图片**
   - 有 "AI-generated" 水印的图片
   - 不同背景颜色
   - 不同分辨率

2. **DALL-E 图片**
   - 有彩色点阵水印的图片
   - 不同点阵密度
   - 不同颜色组合

3. **Midjourney 图片**
   - 有用户名水印的图片
   - 不同文字大小
   - 不同背景对比度

4. **无水印图片**
   - 普通照片
   - 其他 AI 生成图片
   - 验证误报率

### 评估指标

- **准确率** (Accuracy)：正确检测 / 总数
- **召回率** (Recall)：检测到的水印 / 实际有水印
- **精确率** (Precision)：检测到的水印 / 报告有水印
- **F1 分数**：精确率和召回率的调和平均

### 调优方向

1. **调整权重**
   - 根据实际测试结果调整各策略权重
   - 提高准确策略的权重

2. **调整阈值**
   - 根据误报率和漏报率调整
   - 平衡准确性和召回率

3. **添加新策略**
   - 根据失败案例添加新的检测策略
   - 持续改进算法

## 未来改进方向

### 1. OCR 集成

使用 Tesseract OCR 识别文字内容：

```toml
[dependencies]
tesseract = "0.14"
```

优势：
- 直接识别 "AI-generated" 等文字
- 准确率大幅提升
- 可识别更多平台

劣势：
- 需要额外依赖
- 性能开销较大
- 需要训练数据

### 2. 模板匹配

预存储常见水印的模板图片：

```rust

    // 使用归一化互相关 (NCC) 匹配

```

优势：
- 准确率极高
- 速度快
- 不需要复杂算法

劣势：

- 对变形、缩放敏感
- 新水印需要更新模板

使用 CNN 模型进行分类：

```toml

```

优势：
- 准确率最高
- 可学习新模式
- 泛化能力强

劣势：
- 需要训练数据
- 模型文件较大
- 推理速度较慢
- 实现复杂度高

- 需要专业知识

## 总结

新算法通过多策略融合、自适应区域、降低阈值等改进，显著提升了水印检测的准确率和召回率。虽然仍有改进空间（如 OCR、深度学习），但当前算法已经能够满足大部分使用场景。
