# IP 作品集功能需求文档

## 功能概述

作品集（Works Collection）是 IP 资产管理域的扩展功能模块，用于管理作品及其角色信息，并支持将 IP 资产关联到作品中的角色。作品可以是电视剧、电影、小说、话剧、动画、游戏、漫画等任何类型的创作内容。

## 业务目标

1. **扩展 IP 管理能力**：为核心的表情包管理功能提供作品背景信息
2. **独立可用**：作品集可以独立使用，不依赖表情包功能
3. **低耦合设计**：为未来拆分为独立应用（sanWorksBox）做准备
4. **灵活关联**：支持 IP 与作品角色的灵活关联，也支持纯作品角色管理

## 核心需求

### R1: 作品管理

**描述**：用户可以创建、查看、编辑和删除作品信息。

**功能点**：
- R1.1: 创建作品，包含基本信息（名称、类型、描述等）
- R1.2: 查看作品列表，支持网格和列表视图切换
- R1.3: 查看作品详情，包含完整信息和关联的角色列表
- R1.4: 编辑作品信息
- R1.5: 删除作品（软删除）
- R1.6: 为作品设置封面图片
- R1.7: 为作品添加标签（复用现有标签系统）

**验收标准**：
- 可以成功创建包含所有必填字段的作品
- 作品列表正确显示所有未删除的作品
- 作品详情页显示完整信息和关联角色
- 编辑后的作品信息正确保存
- 删除的作品不再显示在列表中（软删除）
- 封面图片正确上传和显示
- 标签正确关联到作品

### R2: 作品类型和状态管理

**描述**：支持多种作品类型和状态跟踪。

**功能点**：
- R2.1: 支持作品类型：电视剧、电影、小说、话剧、动画、游戏、漫画、其他
- R2.2: 支持作品状态：筹备中、制作中、已发布、已完结、已取消
- R2.3: 按作品类型筛选
- R2.4: 按作品状态筛选

**验收标准**：
- 所有作品类型都可以正确选择和保存
- 所有作品状态都可以正确选择和保存
- 筛选功能正确过滤作品列表
- 类型和状态在界面上正确显示（带标签样式）

### R3: 角色管理

**描述**：用户可以在作品中创建、查看、编辑和删除角色。

**功能点**：
- R3.1: 在作品中创建角色，包含角色名称、类型、描述等
- R3.2: 查看作品的角色列表
- R3.3: 编辑角色信息
- R3.4: 删除角色（软删除）
- R3.5: 为角色上传图片（支持多张）
- R3.6: 设置角色的显示顺序（拖拽排序）
- R3.7: 记录角色的出场信息（集数/章节等）

**验收标准**：
- 可以在作品中成功创建角色
- 角色列表正确显示该作品的所有角色
- 角色信息可以正确编辑和保存
- 删除的角色不再显示
- 角色图片正确上传和显示
- 拖拽排序后顺序正确保存
- 出场信息正确保存和显示

### R4: 角色类型管理

**描述**：支持多种角色类型分类。

**功能点**：
- R4.1: 支持角色类型：主角、配角、反派、客串、彩蛋、其他
- R4.2: 角色类型在界面上以标签形式显示
- R4.3: 可以按角色类型筛选（在作品详情页）

**验收标准**：
- 所有角色类型都可以正确选择和保存
- 角色类型标签正确显示
- 筛选功能正确工作

### R5: IP 与角色关联

**描述**：支持将 IP 资产关联到作品中的角色（可选功能）。

**功能点**：
- R5.1: 创建或编辑角色时，可以选择关联一个 IP
- R5.2: 角色可以不关联任何 IP（纯作品角色）
- R5.3: 一个角色只能关联一个 IP
- R5.4: 一个 IP 可以关联到多个作品的不同角色
- R5.5: 删除 IP 时，角色保留但清除 IP 关联（ON DELETE SET NULL）
- R5.6: 在角色卡片上显示关联的 IP 信息（头像、名称）
- R5.7: 点击 IP 信息可以跳转到 IP 详情页

**验收标准**：
- 可以成功将 IP 关联到角色
- 未关联 IP 的角色可以正常创建和显示
- 一个角色只能关联一个 IP（界面限制）
- 同一个 IP 可以关联到多个角色
- 删除 IP 后，角色仍然存在，IP 字段为空
- IP 信息正确显示在角色卡片上
- 跳转到 IP 详情页功能正常

### R6: 作品搜索和筛选

**描述**：用户可以快速找到需要的作品。

**功能点**：
- R6.1: 按作品名称搜索
- R6.2: 按作品类型筛选
- R6.3: 按作品状态筛选
- R6.4: 按标签筛选
- R6.5: 支持多条件组合筛选
- R6.6: 排序选项：按创建时间、更新时间、发布日期、名称

**验收标准**：
- 搜索功能实时过滤作品列表
- 所有筛选条件正确工作
- 多条件组合筛选结果正确
- 所有排序选项正确工作

### R7: 作品详情视图

**描述**：提供完整的作品详情展示和管理界面。

**功能点**：
- R7.1: 左侧显示作品基本信息面板
- R7.2: 右侧显示角色列表面板
- R7.3: 作品信息可以直接编辑（内联编辑）
- R7.4: 角色列表支持添加、编辑、删除操作
- R7.5: 角色卡片可以展开/折叠详细信息
- R7.6: 显示作品的统计信息（角色数量等）

**验收标准**：
- 布局清晰，左右分栏正确显示
- 所有信息正确展示
- 内联编辑功能正常
- 角色操作功能正常
- 展开/折叠动画流畅
- 统计信息准确

### R8: IP 详情页集成（可选）

**描述**：在 IP 详情页中显示该 IP 参与的作品和角色。

**功能点**：
- R8.1: 在 IP 详情页添加"相关作品"标签页
- R8.2: 显示该 IP 扮演的所有角色列表
- R8.3: 角色卡片显示作品信息和角色信息
- R8.4: 点击角色卡片跳转到作品详情页
- R8.5: 通过配置开关控制是否显示此标签页

**验收标准**：
- "相关作品"标签页正确显示
- 角色列表准确显示该 IP 的所有角色
- 跳转功能正常
- 配置开关正确控制显示/隐藏

### R9: 文件存储管理

**描述**：作品和角色的图片文件独立存储。

**功能点**：
- R9.1: 作品封面存储在 `{app_data_dir}/works/{work_id}/cover.{ext}`
- R9.2: 角色图片存储在 `{app_data_dir}/works/{work_id}/characters/{character_id}_{index}.{ext}`
- R9.3: 支持常见图片格式（jpg, png, gif, webp）
- R9.4: 删除作品时清理相关文件
- R9.5: 删除角色时清理相关文件

**验收标准**：
- 图片文件按规定路径存储
- 所有图片格式都能正确处理
- 删除操作正确清理文件
- 不会留下孤立文件

### R10: 数据完整性

**描述**：确保数据的一致性和完整性。

**功能点**：
- R10.1: 软删除支持（作品和角色）
- R10.2: 删除作品时级联删除角色（ON DELETE CASCADE）
- R10.3: 删除 IP 时角色保留但清除关联（ON DELETE SET NULL）
- R10.4: 时间戳自动管理（created_at, updated_at）
- R10.5: 数据验证（必填字段、格式校验）

**验收标准**：
- 软删除的数据可以恢复（如果需要）
- 级联删除正确工作
- IP 删除不影响角色数据
- 时间戳自动更新
- 无效数据被拒绝

### R11: 用户界面

**描述**：提供友好的用户界面。

**功能点**：
- R11.1: 响应式布局，适配不同窗口大小
- R11.2: 加载状态提示
- R11.3: 错误提示和处理
- R11.4: 确认对话框（删除操作）
- R11.5: Toast 通知（操作成功/失败）
- R11.6: 空状态提示（无作品、无角色时）
- R11.7: 图片预览和放大查看

**验收标准**：
- 界面在不同窗口大小下正常显示
- 加载状态清晰可见
- 错误信息准确友好
- 确认对话框在危险操作前显示
- Toast 通知及时准确
- 空状态提示清晰
- 图片预览功能正常

### R12: 性能要求

**描述**：确保功能的性能表现。

**功能点**：
- R12.1: 作品列表加载时间 < 1 秒（100 个作品以内）
- R12.2: 作品详情加载时间 < 500ms
- R12.3: 图片懒加载（列表视图）
- R12.4: 数据库查询优化（适当的索引）
- R12.5: 大图片自动压缩

**验收标准**：
- 加载时间符合要求
- 图片懒加载正常工作
- 数据库查询性能良好
- 大图片不影响性能

## 非功能需求

### NFR1: 低耦合设计

**描述**：作品集功能与 IP 核心功能（表情包）保持低耦合。

**要求**：
- 作品集功能可以完全禁用而不影响表情包功能
- 数据库表独立，仅通过外键松耦合
- 后端命令模块独立
- 前端组件和 Store 独立
- 文件存储独立

**验收标准**：
- 可以通过配置开关禁用作品集功能
- 删除作品集相关代码不影响其他功能
- 数据库表可以独立迁移

### NFR2: 可扩展性

**描述**：为未来功能扩展预留空间。

**要求**：
- 数据模型支持扩展字段
- 接口设计考虑向后兼容
- 支持插件机制（未来）

**验收标准**：
- 可以添加新字段而不破坏现有功能
- API 版本化或向后兼容

### NFR3: 数据安全

**描述**：确保用户数据的安全性。

**要求**：
- 所有数据存储在本地
- 文件路径安全验证
- SQL 注入防护
- 输入数据验证和清理

**验收标准**：
- 无数据泄露风险
- 路径遍历攻击防护
- SQL 查询安全
- 恶意输入被拒绝

### NFR4: 用户体验

**描述**：提供流畅的用户体验。

**要求**：
- 界面响应时间 < 100ms
- 动画流畅（60fps）
- 操作反馈及时
- 错误恢复友好

**验收标准**：
- 界面操作流畅无卡顿
- 动画无掉帧
- 用户操作有即时反馈
- 错误后可以继续操作

## 约束条件

1. **技术栈**：必须使用 Tauri 2.0 + React + Rust
2. **数据库**：使用 SQLite（rusqlite）
3. **UI 组件**：使用 shadcn/ui 组件库
4. **状态管理**：使用 Zustand
5. **兼容性**：支持 Windows 10/11
6. **语言**：界面语言为简体中文

## 未来扩展

以下功能不在当前版本范围内，但为未来扩展预留：

1. **时间线视图**：按时间轴展示作品发布历史
2. **关系图谱**：可视化 IP 与作品之间的关系网络
3. **协作功能**：多人协作编辑作品信息
4. **版本控制**：跟踪作品信息的修改历史
5. **导出功能**：导出作品信息为 PDF/Word 文档
6. **统计分析**：IP 出演频率、作品类型分布等
7. **外部集成**：与豆瓣、IMDb 等平台集成获取作品信息
8. **表情包关联**：表情包可以标注来源作品和角色

## 验收测试场景

### 场景 1: 创建完整的作品
1. 用户打开作品集视图
2. 点击"新建作品"按钮
3. 填写作品信息（名称、类型、描述等）
4. 上传封面图片
5. 添加标签
6. 保存作品
7. 验证作品出现在列表中

### 场景 2: 为作品添加角色
1. 用户打开作品详情页
2. 点击"添加角色"按钮
3. 填写角色信息（名称、类型、描述等）
4. 上传角色图片
5. 保存角色
6. 验证角色出现在角色列表中

### 场景 3: 关联 IP 到角色
1. 用户编辑一个角色
2. 在 IP 选择器中选择一个 IP
3. 填写关联说明
4. 保存角色
5. 验证角色卡片显示 IP 信息
6. 点击 IP 信息跳转到 IP 详情页

### 场景 4: 搜索和筛选作品
1. 用户在搜索框输入作品名称
2. 验证列表实时过滤
3. 选择作品类型筛选
4. 验证列表正确过滤
5. 选择排序方式
6. 验证列表正确排序

### 场景 5: 删除作品
1. 用户选择一个作品
2. 点击删除按钮
3. 确认删除对话框出现
4. 确认删除
5. 验证作品从列表中消失
6. 验证相关文件被清理
